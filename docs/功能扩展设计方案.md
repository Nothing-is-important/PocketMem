# PocketMemory 功能扩展设计方案

> **设计原则：** 每次扩展只加"这个场景真正需要的"，不做通用框架式的过度设计。
> **参考来源：** MewCode Agent 课程架构 + PocketMemory 实际场景需求
> **创建日期：** 2026-05-30

---

## 总览

```
现有系统
├── Agent Loop (Router→Retrieve→Judge→Generate→Reflect) ← 不改
├── 检索算子 (向量/BM25/实体/时间)                        ← 不改
├── InferenceBackend 抽象层                              ← 不改
```

## 扩展 1：System Prompt 集中管理

### 1.1 现状 vs 目标

**现状：** Prompt 散落在代码中

```python
# agent/router.py 第 15 行 —— 硬编码
ROUTER_PROMPT = """分析用户查询意图，只回复一个词：..."""

# agent/generator.py 第 7 行 —— 硬编码
GENERATOR_SYSTEM_PROMPT = """你是一个个人记忆助手。..."""

# agent/memory_judge.py 第 22 行 —— 硬编码
MEMORY_JUDGE_PROMPT = """你是一个个人记忆助手。判断..."""
```

**目标：** Prompt 集中在配置文件中，每个有版本号和迭代记录

```yaml
# config/prompts/router.yaml
version: 2
last_updated: 2026-05-29
changelog:
  - v1: "JSON 输出格式 → 1.5B 模型 JSON 不稳定，解析成功率 70%"
  - v2: "改为单字输出 + 关键词兜底 → 解析成功率 95%+兜底到 100%"
prompt: |
  分析用户查询意图，只回复一个词：memory_lookup、knowledge_lookup、general 或 refuse。
  ...
```

### 1.2 文件结构

```
config/prompts/
├── __init__.py          # PromptLoader 类
├── router.yaml          # Router 意图分类
├── generator.yaml       # 答案生成 system prompt
├── generator_context.yaml  # 带上下文的生成模板
├── judge_rerank.yaml    # MemoryJudge LLM 精排（当前禁用，保留模板）
├── judge_reflect.yaml   # Reflect 补充查询生成
└── benchmark.yaml       # LLM-as-Judge 评分
```

### 1.3 核心类设计

```python
# config/prompts/__init__.py

import yaml
from pathlib import Path
from typing import Dict

class PromptLoader:
    """集中管理所有 LLM Prompt，支持版本记录和热加载。"""

    def __init__(self, prompts_dir: str = "config/prompts"):
        self._dir = Path(prompts_dir)
        self._cache: Dict[str, dict] = {}
        self._load_all()

    def _load_all(self):
        """启动时加载所有 prompt 文件。"""
        for yaml_file in self._dir.glob("*.yaml"):
            name = yaml_file.stem
            with open(yaml_file, "r", encoding="utf-8") as f:
                self._cache[name] = yaml.safe_load(f)

    def get_prompt(self, name: str) -> str:
        """获取 prompt 文本。"""
        return self._cache[name]["prompt"]

    def get_version(self, name: str) -> int:
        """获取当前版本号。"""
        return self._cache[name]["version"]

    def get_changelog(self, name: str) -> list:
        """获取迭代历史（面试用）。"""
        return self._cache[name].get("changelog", [])

    def reload(self):
        """热加载——改 prompt 不需要重启服务。"""
        self._cache.clear()
        self._load_all()

# 全局单例
prompts = PromptLoader()
```

## 扩展 2：Hook 生命周期系统

### 2.1 设计思路

**不是**做一个通用的、支持任意事件的 Hook 系统。**而是**在 Agent 图的 4 个关键节点预留挂载点，后续扩展通过注册 hook 函数接入，不改核心代码。

### 2.2 Agent 执行流程中的 4 个挂载点

```
用户查询
   │
   ▼
┌─────────────────┐
│  pre_route       │ ← Hook 1: 查询前的预处理
│  (查询改写?)     │    例如：拼写纠错、查询扩展
└────────┬────────┘
         ▼
      Router（意图分类）
         │
         ▼
┌─────────────────┐
│  pre_retrieve    │ ← Hook 2: 检索前的准备
│  (缓存检查?)     │    例如：检查缓存、注入用户画像
└────────┬────────┘
         ▼
      Retriever（检索）
         │
         ▼
┌─────────────────┐
│  post_retrieve   │ ← Hook 3: 检索后的增强
│  (结果过滤?)     │    例如：时间衰减、敏感词过滤、多样性重排
└────────┬────────┘
         ▼
    MemoryJudge（判断）
         │
         ▼
    Generator（生成）
         │
         ▼
┌─────────────────┐
│  post_generate   │ ← Hook 4: 生成后的处理
│  (记录日志?)     │    例如：更新用户画像、记录访问频率、缓存结果
└────────┬────────┘
         ▼
      返回答案
```

### 2.3 核心类设计

```python
# agent/hooks.py

from typing import Any, Callable, Dict, List

# Hook 函数签名
HookFunc = Callable[[Dict[str, Any]], Dict[str, Any]]

class HookRegistry:
    """轻量级 Hook 注册中心。

    设计理念：
    - 不定义事件类型——4 个挂载点就是 4 个注册表
    - 每个挂载点可以注册多个 hook，按注册顺序执行
    - Hook 函数接收 state dict，返回 state dict（管道模式）
    - 失败的 hook 不阻塞主流程（记录日志后继续）
    """

    def __init__(self):
        self._hooks: Dict[str, List[HookFunc]] = {
            "pre_route": [],
            "pre_retrieve": [],
            "post_retrieve": [],
            "post_generate": [],
        }

    def register(self, point: str, hook: HookFunc):
        """注册一个 hook 到指定挂载点。"""
        if point not in self._hooks:
            raise ValueError(f"Unknown hook point: {point}. "
                             f"Available: {list(self._hooks.keys())}")
        self._hooks[point].append(hook)

    def run(self, point: str, state: dict) -> dict:
        """执行指定挂载点的所有 hook。"""
        for hook in self._hooks.get(point, []):
            try:
                state = hook(state)
            except Exception as e:
                # Hook 失败不阻塞主流程
                from utils import get_logger
                get_logger("hooks").warning(
                    "Hook %s at %s failed: %s", hook.__name__, point, e
                )
        return state


# 全局单例
hooks = HookRegistry()
```

在 `agent/graph.py` 的 `build_agent_graph()` 中，修改各节点函数使其在执行前后调用 hook：

### 2.6 预定义的 Hook 示例

```python
# 内置 hook（可以作为默认注册的示例）
def hook_log_query(state: dict) -> dict:
    """记录查询日志（pre_route hook）"""
    from datetime import datetime
    state.setdefault("_log", {})["query_time"] = datetime.now().isoformat()
    return state

def hook_apply_temporal_decay(state: dict) -> dict:
    """应用时间衰减（post_retrieve hook）—— 把现有逻辑搬进 hook"""
    from agent.retriever_node import apply_temporal_decay
    if state.get("intent") == "memory_lookup":
        state["retrieval_results"] = apply_temporal_decay(
            state["retrieval_results"]
        )
    return state

def hook_update_user_profile(state: dict) -> dict:
    """更新用户画像（post_generate hook）—— 扩展 3 实现"""
    from agent.user_profile import profile
    profile.record_query(state.get("query", ""))
    return state
```

### 2.7 面试讲法

> "我设计了一个轻量级的 Hook 系统——不需要 MewCode 那样完整的事件总线，只在 Agent 执行的 4 个关键节点（路由前、检索前、检索后、生成后）预留了挂载点。每个挂载点可以注册多个 hook，按管道模式串联。后续加功能——比如用户画像、查询日志、敏感词过滤——只需要写一个 hook 函数注册进去，不改 Agent 核心代码。这体现了'对扩展开放，对修改关闭'的设计原则。"

## 扩展 3：跨会话记忆 + 用户画像

### 3.1 现状 vs 目标

**现状：**
- ChromaDB 存储聊天记录（数据持久）
- 但系统不知道用户是谁、关心什么、之前搜过什么
- 每次重启，"上次那个火锅店"中的"上次"没有上下文

**目标：**
- 系统能记住用户的查询偏好（经常搜什么、关注谁）
- 系统能记住哪些记忆片段被频繁访问（重要内容自动加权）
- 用户画像作为 pre_generate hook 注入到 prompt 中，让回答更个性化

### 3.2 数据结构

```python

{

    "total_queries": 156,
    "last_session": "2026-05-29T18:30:00",

      "张三": 23,
      "火锅店": 12,

      "general": 13
    }
  },
  "contact_importance": {     # 联系人关注度（根据查询提及频率）
    "张三": {"mention_count": 34, "last_queried": "2026-05-29"},
    "李四": {"mention_count": 18, "last_queried": "2026-05-28"},
    "王五": {"mention_count": 7,  "last_queried": "2026-05-15"}

  },

    "chunk_abc123": {"access_count": 12, "last_accessed": "2026-05-29"},
```

**存储位置：** `~/.pocket_memory/user_profile.json`（纯 JSON，不依赖数据库）

### 3.3 核心类设计

```python
# agent/user_profile.py

import json
from pathlib import Path
from collections import Counter
from typing import Dict, List, Optional

class UserProfile:
    """用户画像管理器。

    通过 post_generate hook 自动记录查询行为，
    通过 pre_generate hook 注入用户画像到生成上下文。
    """

    def __init__(self, profile_path: str = "~/.pocket_memory/user_profile.json"):
        self._path = Path(profile_path).expanduser()
        self._data = self._load()
        self._pending_updates = False

    # ── 记录 ──

    def record_query(self, query: str, intent: str = ""):
        """记录一次查询（在 post_generate hook 中调用）。"""
        self._data["query_stats"]["total_queries"] += 1

        # 提取高频词（用 jieba 分词 + 停用词过滤）
        words = self._extract_keywords(query)
        freq = self._data["query_stats"]["frequent_terms"]
        for w in words:
            freq[w] = freq.get(w, 0) + 1

        if intent:
            intents = self._data["query_stats"]["frequent_intents"]
            intents[intent] = intents.get(intent, 0) + 1

        self._pending_updates = True

    def record_access(self, chunk_id: str):
        """记录一次记忆片段访问。"""
        history = self._data["access_history"]
        if chunk_id not in history:
            history[chunk_id] = {"access_count": 0}
        history[chunk_id]["access_count"] += 1
        from datetime import datetime
        history[chunk_id]["last_accessed"] = datetime.now().isoformat()
        self._pending_updates = True

    # ── 注入 ──

    def inject_context(self, query: str) -> str:
        """生成用户画像上下文文本，注入到 generator prompt。

        Returns:
            一段描述用户画像的文本，如：
            "用户经常查询与「张三」(23次)、「火锅店」(12次) 相关的内容。
             最近关注话题：技术学习、美食。"
        """
        parts = []

        # 高频查询词
        freq = self._data["query_stats"]["frequent_terms"]
        if freq:
            top_terms = sorted(freq.items(), key=lambda x: x[1], reverse=True)[:5]
            terms_str = "、".join(f"「{t}」({c}次)" for t, c in top_terms)
            parts.append(f"用户经常查询与 {terms_str} 相关的内容。")

        # 常用联系人
        contacts = self._data["contact_importance"]
        if contacts:
            top_contacts = sorted(
                contacts.items(),
                key=lambda x: x[1]["mention_count"],
                reverse=True
            )[:3]
            contacts_str = "、".join(name for name, _ in top_contacts)
            parts.append(f"用户常提及的人：{contacts_str}。")

        return "\n".join(parts) if parts else ""

    # ── 持久化 ──

    def save(self):
        """保存用户画像到磁盘（定期调用，避免每次查询都写盘）。"""
        if not self._pending_updates:
            return
        self._path.parent.mkdir(parents=True, exist_ok=True)
        with open(self._path, "w", encoding="utf-8") as f:
            json.dump(self._data, f, ensure_ascii=False, indent=2)
        self._pending_updates = False

    def _load(self) -> dict:
        if self._path.exists():
            with open(self._path, "r", encoding="utf-8") as f:
                return json.load(f)
        return self._default_profile()

    def _default_profile(self) -> dict:
        return {
            "query_stats": {
                "total_queries": 0,
                "last_session": "",
                "frequent_terms": {},
                "frequent_intents": {},
            },
            "contact_importance": {},
            "topic_preferences": {},
            "access_history": {},
        }

    def _extract_keywords(self, text: str) -> List[str]:
        """从查询文本中提取关键词（轻量版）。"""
        import jieba
        words = jieba.lcut(text)
        # 过滤：长度 >=2，不是纯标点/数字
        return [
            w for w in words
            if len(w) >= 2
            and not all(c in "，。！？；：""''（）【】《》…—·0123456789" for c in w)
        ]


# 全局单例
profile = UserProfile()
```

### 3.4 集成方式——零侵入 Agent 核心代码

通过 Hook 系统接入，不改 Agent 图：

```python
# scripts/run_demo.py setup() 中注册

from agent.hooks import hooks
from agent.user_profile import profile

# 注册 post_generate hook：每次查询后更新用户画像
hooks.register("post_generate", lambda state: (
    profile.record_query(state.get("query", ""), state.get("intent", "")),
    profile.save(),
    state
)[-1])

# 注册 pre_retrieve hook：检索前注入用户画像
def inject_profile(state):
    context = profile.inject_context(state.get("query", ""))
    if context:
        state["user_context"] = context
    return state
hooks.register("pre_retrieve", inject_profile)
```

### 3.6 面试讲法

> "我觉得一个真正的记忆系统不应该在每次重启后'失忆'。所以加了一层用户画像——系统在后台默默观察用户的查询行为，自动提取高频查询词、关注话题和重要联系人。这些信息在下次查询时作为上下文注入到 prompt 里——用户问'上次那个火锅店'时，系统已经从画像里知道'火锅店'是高频话题，检索和回答都更有针对性。
>
> 这个功能的巧妙之处在于它完全通过我之前设计的 Hook 系统接入——不改 Agent 核心代码，只是注册了两个 hook 函数。这证明了 Hook 架构的价值：新功能可以像插件一样'插'进现有系统。"
