# PocketMemory 深度记忆系统设计方案

## 一句话定位

不是"能搜索聊天记录的 RAG"，而是**用 AI 建模你的个人记忆——它会遗忘、会巩固、会联想、会主动提醒。**

---

## Part 1：为什么现在的设计"浅"

对比一下：

```
人类记忆做的事情                    PocketMemory 现在做的
─────────────────────────────────────────────────────
记住重要的事，忘记琐事              所有消息同等对待
反复回忆的东西越来越牢              没有访问频率反馈
一段经历会浓缩成一个结论            原始消息不变，不摘要
提到"火锅"会自动想起"张三"          chunk 之间互不关联
"好久没联系张三了"有感知            完全被动，不主动
昨天的具体对话→一个月后变概括        无时间维度的抽象
```

**一句话：目前的系统是个搜索引擎，不是记忆系统。**

## Part 2：记忆的三个层次

```
┌─────────────────────────────────────────────────────────┐
│ Layer 3: 长期记忆 (Semantic Memory)                     │
│ - 跨会话提取的事实和知识                                  │
│ - 自动摘要合并："张三推荐渝味火锅，朝阳区，电话138xxx"     │
│ - 持久存储，低衰减                                       │
│ - 可以被直接回答，不需要引用原始消息                       │
├─────────────────────────────────────────────────────────┤
│ Layer 2: 短期记忆 (Episodic Memory)                     │
│ - 完整的会话片段（几天到几周）                            │
│ - 时间衰减 + 访问频率反馈                                │
│ - 被多次搜索 → 强化 → 提升到长期记忆                      │
├─────────────────────────────────────────────────────────┤
│ Layer 1: 感官记忆 (Sensory Buffer)                      │
│ - 最近 24h 的原始消息                                    │
│ - 快速衰减（可能被注意机制筛选）                          │
│ - 被回复/被引用 → 进入短期记忆                            │
└─────────────────────────────────────────────────────────┘
```

**这和我们现在有什么不同？** 现在就一层——所有消息放 ChromaDB，无差别对待。新架构是**三层生命周期管理**。

---

## Part 3：核心机制详解

### 3.1 遗忘曲线——个性化、多类型

现在的固定 30 天半衰期，问题在哪？**不同的记忆衰减速度完全不同。**

```
消息类型                  半衰期      理由
──────────────────────────────────────────────
"好的" "收到"            1天        日常应答，无记忆价值
"周末聚餐？"             7天        社交规划，短期有用
"offer拿到了！"          90天       情绪事件，记得更久
"渝味火锅 朝阳区 138xxx" 180天      包含具体信息的决策
"Transformer架构笔记"    365天      知识型记忆，长期保留
```

**实现方式：** 不需要用 LLM 判断每条消息——用信号：

- 被回复了？（interaction_signal = 1.2x）
- 被搜索了 N 次？（access_count → 强化）
- 包含日期/地点/数字？（info_density → 慢衰减）
- 包含感叹号/情绪词？（emotional_marker → 慢衰减）
- 消息长度 > 50 字？（高信息量 → 慢衰减）

**面试关键点：** "我没有用单一的遗忘曲线，而是基于信息类型自适应调整半衰期——这比固定衰减的 Recall@5 提升了 X%。"

### 3.2 记忆巩固——从片段到知识

这是这个设计最核心的差异化能力。

**现状：** 用户问"张三推荐了哪家火锅店？"→ 返回原始聊天记录片段。

**应该：** 系统自动把相关对话合并成一个**知识卡片**：

```
原始消息（一周内）：
  Day1: "周末去渝味火锅吧，在朝阳区建国路88号"
  Day3: "渝味火锅电话138xxxx，记得订位"
  Day5: "周六中午12点，叫上李四"

记忆巩固后（一周后）：  
  ┌─────────────────────────────┐
  │ 张三推荐：渝味火锅           │
  │ 地址：朝阳区建国路88号        │
  │ 电话：138xxxx               │
  │ 聚餐日期：周六中午12点        │
  │ 参与人：张三、我、李四        │
  │ 最近一次提到：6月3日          │
  │ [展开原始对话]               │
  └─────────────────────────────┘
```

**实现方式：** 离线 Consolidation Worker。

每天或每周跑一次，扫描最近 7 天的对话片段，用 LLM 检测哪些片段在讨论**同一个话题**（用话题聚类），然后合并成一张记忆卡片。原始消息保留（可展开），但检索时优先返回卡片。

**面试关键点：** "我设计了记忆巩固机制——自动把关于同一话题的多轮对话合并成结构化知识卡片。这不只是检索，是模拟人类记忆从 episodic 到 semantic 的转化过程。"

### 3.3 关联记忆网络

**现状：** 搜索"火锅"→ 返回包含"火锅"的 chunk。搜索"张三"→ 返回张三的对话。两个查询之间没有关联。

**应该：** 实体之间自动建图。

```
              ┌──────────┐
              │  渝味火锅  │
              └────┬─────┘
       推荐        │        位于
    ┌──────────────┼──────────────┐
    ▼              ▼              ▼
┌──────┐     ┌──────────┐    ┌──────────┐
│ 张三  │────│  3月15日  │    │ 朝阳区    │
└──┬───┘     └──────────┘    └────┬─────┘
   │ 同事                        │
   │                            │ 也去过
   ▼                            ▼
┌──────┐                    ┌──────────┐
│ 李四  │─────────────────→│ 建国路88号│
└──────┘                    └──────────┘
```

查询"张三"时，不只是找到张三的消息，还能**自动关联**到渝味火锅、朝阳区。查询"朝阳区有什么好吃的"→ 自动关联到渝味火锅→张三推荐的。

**实现方式：** 
- 轻量级实现：社区检测算法（如 Louvain）对实体共现做聚类
- 存储：ChromaDB 元数据 + 一个简单的 SQLite 图关联表
- 不需要 Neo4j！这个规模用 SQLite 完全够

**面试关键点：** "我构建了一个关联记忆网络——当用户查询一个实体时，系统会自动激活关联实体，模拟人类记忆的语义启动效应。实现上不需要知识图谱数据库，SQLite 加实体共现分析就够——这体现了约束驱动的技术选型。"

### 3.4 访问频率反馈

**现状：** 时间衰减是单向的——只会越来越低。

**应该：** 被搜索到的记忆应该被**强化**。

```
记忆强度 = 基础强度 × exp(-λ × 天数) + 访问次数 × 0.1
                                         ↑
                              每次被成功检索，加0.1
                              三次搜索"火锅店"→记忆变强→衰减更慢
```

**面试关键点：** "我加入了访问频率反馈机制——这和人类记忆的测试效应完全一致。被反复回忆的记忆会变得更牢固，衰减更慢。这比单纯的时间衰减在召回率上提升了 X%。"

### 3.5 主动记忆提醒

**现状：** 完全被动——用户不搜，系统不动。

**应该：** 系统应该主动提醒。

```
触发场景：
- "你一周没和张三联系了，上次他推荐了渝味火锅"
- "下周三就是和李四讨论的那个项目 deadline"
- "你上个月搜过 3 次 Transformer 架构，最近有篇新论文"
- "周末快到了，上次团建你投了古北水镇，需要确认吗？"
```

**实现方式：** 一个轻量级的 Check-in Worker，每次系统启动时扫描：
1. 高重要性 + 即将到期的记忆（deadline 提醒）
2. 频繁搜索但最近未更新的主题
3. 长时间未联系但历史互动频繁的联系人

**面试关键点：** "我让系统从被动检索升级为主动服务——这是个人 AI 助手的自然进化方向。Siri 和 Alexa 也会主动提醒，但它们靠的是规则引擎；PocketMemory 的主动提醒基于记忆的重要性评分和访问模式分析。"

## Part 4：架构变更

```
现在的架构：
  Router → Retrieve → Judge → Generate

新架构：
  Router → ThreeLayerRetrieve ─→ Consolidate? → Generate
              │                      │
     ┌────────┼────────┐             │ (每天/每周触发)
     ▼        ▼        ▼             ▼
  Sensory  Episodic Semantic    Memory Cards
  (<24h)  (days)   (forever)    (LLM 摘要合并)
     │        │        │
     └────────┼────────┘
              ▼
        MemoryGraph
     (实体关联激活)
              │
              ▼
        Judge (访问频率纳入评分)
              │
              ▼
        Generate + 关联实体上下文
```

**新增模块：**

| 模块 | 功能 | 复杂度 |
|------|------|--------|
| `memory_layers.py` | 三层记忆生命周期管理 | 中 |
| `memory_consolidator.py` | 离线记忆巩固 Worker | 高 |
| `memory_graph.py` | 实体关联网络（SQLite） | 中 |
| `forgetting_curve.py` | 自适应遗忘曲线 | 低 |
| `access_tracker.py` | 访问频率追踪 | 低 |
| `proactive_checker.py` | 主动记忆提醒 | 低 |

## Part 5：面试叙事

这个方向面试时是一条完整的叙事线：

**开场（场景共鸣）：**
> "你有没有这种感觉——你和朋友的聊天记录里有大量有价值的信息，但微信搜索只能搜关键词。你说'上次张三推荐的那家火锅店'，它搜不到；你说'团建最后定了什么方案'，它也搜不到。我想解决的就是这个问题——一个真正像人类记忆的 AI 系统。"

**技术纵深（5 分钟独白）：**
> "我设计了三个核心机制。第一是三层记忆生命周期——感官记忆 24 小时内衰减、短期记忆按信息重要性自适应遗忘曲线、长期记忆通过 LLM 摘要巩固成为结构化的知识卡片。第二是关联记忆网络——通过实体共现自动构建人-地点-事件的关联图，查询一个节点自动激活关联节点，模拟人类记忆的语义启动效应。第三是访问频率反馈——被多次搜索的记忆会强化，这和认知科学中的测试效应一致。"

**接地气（展示数据）：**
> "我用这三个机制做了一个实验：拿自己 3 个月的微信聊天记录，对比有记忆巩固之前和之后的检索效果。'张三推荐了什么'这类复合实体查询的 MRR 从 XX 提升到 XX。而且因为记忆卡片是结构化的，LLM 可以直接引用卡片里的电话号码和地址，幻觉率降低 XX%。"

**引导到擅长方向：**
> "这个系统的底层还是端侧推理——BGE Embedding 做检索，Qwen 做生成，全部跑在 4060 上。我在实习里做的模型量化和推理优化，在这个项目里直接落地了。"

## Part 6：实现优先级

不是所有东西都要做。按面试价值排序：

| 优先级 | 特性 | 面试说服力 | 工作量 |
|--------|------|-----------|--------|
| P0 | 自适应遗忘曲线 | 高（有数据对比） | 半天 |
| P0 | 访问频率反馈 | 高（认知科学引用） | 半天 |
| P1 | 关联记忆网络 | 最高（差异化的核心） | 1.5天 |
| P1 | 记忆巩固摘要 | 最高（记忆系统 vs 搜索引擎） | 2天 |
| P2 | 三层记忆分层 | 中（架构层面） | 1天 |
| P3 | 主动记忆提醒 | 低（锦上添花） | 1天 |

**建议：P0 + P1 两周搞定，面试就能讲 15 分钟不重样。**
